# 🚀 프로젝트 소개

**하이퍼캐주얼형 쿠킹 미니게임**

'그 때 그 게임'이라는 공모전 주제에 맞추어 '슈의 라면가게'에서 라면이 키오스크와 라면기계로 만들어진다면? 이라는 발상으로 구상을 시작한 게임입니다.  

키오스크로 하는 주문은 편리할 수도 있지만 누군가에게는 느리거나 불편하게만 느껴지기도 합니다.
그런 사람들을 생각하며 만들었습니다.  

도움이 필요한 손님들을 클릭하고, 필요한 재료를 드래그하여 냄비에 넣습니다.  
필요 없는 재료를 넣었다면 다시 드래그하여 냄비 밖으로 뺄 수 있습니다.  
알맞은 재료를 모두 넣고 "재료 전달" 버튼을 누르면 라면 완성!  
목표 금액을 넘는 매출을 올려 할머니에게 인정받고 용돈을 법시다!

# 좋았던 점
서로 긴밀하게 소통하며 각자의 코드를 충분히 이해하는 과정을 중요시한 프로젝트였습니다. 
- 노션 및 옵시디언을 활용하여 각 기능을 어떻게 개발하고, 어떤 생각으로 만들었는지를 공유하는 과정을 거쳤습니다.
- 기능 개발 후 Pull Request 및 코드리뷰 과정을 거쳐 좀 더 견고한 코드를 만들고자 하였습니다

이렇게 한 이유는 현업과정을 그대로 차용해보고자 한 의도도 있습니다만, 가장 큰 이유는 서로 작업시간이 달라 각자의 코드를 고칠 일이 많았기 때문입니다. 서로의 코드를 잘 알고 있어야만 실수없이 고칠 수 있기에 이러한 과정을 거쳤습니다. 물론 바쁠 때는 융통성 있게 생략하기도 했습니다만 대부분의 기간을 이렇게 개발하였습니다.

# 아쉬운 점
 뒤늦은 피드백 후 개발된 기능 중 절반 정도를 덜어내어야 했습니다. 다른 사람들의 피드백을 듣고 수정하는 작업을 너무 늦게 진행했습니다. 
- 기능을 모두 개발해놓고 테스트를 하는 것보다, 핵심기능을 먼저 개발하고 이후 개발을 이어서 했어야 했습니다. 
- 개발 중 완벽한 상태로 보여주고 싶다는 욕심에 전자로 생각이 흘러가버린 점이 아쉽습니다

다음 개발에는 핵심 기능을 먼저 개발하는 것에 초점을 맞추어 빠른 피드백을 통해 작업 방향을 잡을 예정입니다.
